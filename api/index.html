Introduction

The Auth Server API provides user registration and authentication features.

The authentication method used to make requests is JSON Web Token. After a successful registration and email verification users can log in with their credentials. Then the server responds with an access_token.

The client must send the token in the Authorization header when making protected requests.

Authorization: Bearer access_token

API

/register

Register the user.


Request

Field Type Description
username string The user's username.
email string The user's email.
password string The user's password.
avatar string The user's avatar URL.
name string The user's name.
surname string The user's surname.
birthdate string The user's birthdate (yyyy-MM-dd).
role number The user's role.

Response

Field Type Description
id number The user's ID.
username string The user's username.
email string The user's email.
avatar string The user's avatar URL.
name string The user's name.
surname string The user's surname.
birthdate string The user's birthdate (yyyy-MM-dd).
role number The user's role.
verified boolean Is the user's email verified?

/verify/:token

Verify the user's email.


Request

This request doesn't have a body.


Response

The response values are true or false.

Errors

The API uses the following errors:

Error Status Description
100 500 Generic error.
101 404 Invalid request.
102 400 Validation error.
103 409 Username is already being used.
104 409 Email is already being used.
105 401 Email not verified.
106 401 Invalid password.
107 401 Invalid token.
108 404 User not found.
109 503 Mail service error.

For more information check the list of HTTP status codes.
